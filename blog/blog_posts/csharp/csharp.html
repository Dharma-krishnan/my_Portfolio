C# programming fundamentals

What is c# ?


C# is an object-oriented language, which means it supports the principles of encapsulation, inheritance, and polymorphism. Developers use classes and objects to structure their code.

Data Types,


Here are the common data types in C#:
  • Value Types:
    • byte: 8-bit unsigned integer (0 to 255).
    • sbyte: 8-bit signed integer (-128 to 127).
    • short: 16-bit signed integer (-32,768 to 32,767).
    • ushort: 16-bit unsigned integer (0 to 65,535).
    • int: 32-bit signed integer (-2,147,483,648 to 2,147,483,647).
    • uint: 32-bit unsigned integer (0 to 4,294,967,295).
    • long: 64-bit signed integer (-9,223,372,036,854,775,808 to 9,223,372,036,854,775,807).
    • ulong: 64-bit unsigned integer (0 to 18,446,744,073,709,551,615).
    • float: 32-bit single-precision floating-point number.
    • double: 64-bit double-precision floating-point number.
    • decimal: 128-bit decimal floating-point number (useful for financial and monetary calculations).
    • bool: Represents a Boolean value (true or false).
    • char: Represents a single Unicode character.
    • enum: Allows you to define a set of named constant values.
    • struct: User-defined value types that can contain fields, properties, and methods.
  • Reference Types:
    • class: User-defined reference types.
    • string: Represents a sequence of Unicode characters.
    • object: The base type for all types in C#.
    • interface: Defines a contract that classes can implement.
  • Nullable Value Types:
    • These are a special category that allows value types to represent null values. For example, int? can represent an int or null.
  • Custom Data Types:
    • Developers can define their own custom data types using classes, structs, and enums.
  • Dynamic Type:
    • Introduced in C# 4.0, the dynamic keyword allows you to create variables whose types are determined at runtime. It provides more flexibility but sacrifices compile-time type checking.
  • Pointer Types (Unsafe Code):
    • C# allows you to work with pointers, but this feature is generally used in low-level programming and requires the use of the unsafe keyword.
